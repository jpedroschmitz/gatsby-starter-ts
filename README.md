## Quick Start

The best way to start with this template is using the [Gatsby CLI](https://www.gatsbyjs.com/docs/reference/gatsby-cli/).

```
npx gatsby new starter-ts https://github.com/jpedroschmitz/gatsby-starter-ts
```

### Development

To start the project locally, run:

```bash
yarn start
```

Open `http://localhost:8000` with your browser to see the result.

## Documentation

### Requirements

- Node.js >= 14.17
- Yarn 1 (Classic)

### Directory Structure

- [`__helpers__`](./__helpers__/) â€” Helpers files for testing configuration.<br>
- [`__mocks__`](./__mocks__/) â€” Mocks for testing.<br>
- [`.github`](.github) â€” GitHub configuration including the CI workflow.<br>
- [`.husky`](.husky) â€” Husky configuration and hooks.<br>
- [`src`](./src) â€” Application source code, including pages, components, styles.

### Scripts

- `yarn start` â€” Starts the application in development mode at `http://localhost:8000`.
- `yarn build` â€” Compile your application and make it ready for deployment.
- `yarn serve` â€” Serve the production build of your site
- `yarn clean` â€” Wipe out the cache (`.cache` folder).
- `yarn type-check` â€” Validate code using TypeScript compiler.
- `yarn lint` â€” Runs ESLint for all files in the `src` directory.
- `yarn format` â€” Runs Prettier for all files in the `src` directory.
- `yarn commit` â€” Run commitizen. Alternative to `git commit`.
- `yarn test` â€” Run tests.

### Path Mapping

TypeScript are pre-configured with custom path mappings. To import components or files, use the `@` prefix.

```tsx
import { Button } from '@/components/Button';

// To import images or other files from the static folder
import avatar from '@/static/avatar.png';
```

### Switch to npm

This starter uses Yarn 1 (Classic) by default, but this choice is yours. If you'd like to switch to npm, delete the `yarn.lock` file, install the dependencies with `npm i`, change the CI workflow, and Husky Git hooks to use npm commands.
